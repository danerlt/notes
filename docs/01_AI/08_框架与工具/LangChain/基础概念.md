# LangChain 基础概念

## LangChain 简介与核心价值

LangChain 是一个用于构建基于大型语言模型（LLM）应用的开源框架，由 Harrison Chase 于 2022 年发布。它的核心价值在于将 LLM 与外部数据源、工具和计算能力连接起来，让开发者能够构建具有记忆、推理和行动能力的复杂 AI 应用。

### 核心价值

- **标准化接口**：统一封装不同 LLM 提供商（OpenAI、Anthropic、Ollama 等），切换模型只需修改几行代码
- **组件化设计**：提示词、模型、输出解析等模块可以自由组合
- **生态丰富**：内置数百种工具集成（搜索、数据库、API 等）
- **可观测性**：通过 LangSmith 实现链路追踪和调试

### 适用场景

- 聊天机器人与对话助手
- 文档问答（RAG 系统）
- 智能 Agent（自主规划与工具调用）
- 数据提取与结构化输出
- 代码生成与分析

---

## 核心组件概览

LangChain 的架构由以下几个核心组件构成：

```
Model I/O  →  Chains  →  Agents
    ↑              ↑          ↑
PromptTemplate  Memory   Tools/Callbacks
```

### Model I/O

负责与语言模型交互的标准化层，包含三部分：
- **PromptTemplate**：构建动态提示词
- **Language Model**：封装 LLM 调用（LLM / ChatModel）
- **OutputParser**：将模型输出解析为结构化数据

### Chains（链）

将多个组件串联成一个可执行流程。最简单的 Chain 就是 `PromptTemplate → LLM → OutputParser`。

### Agents（智能体）

让 LLM 自主决定调用哪些工具、以什么顺序执行，实现复杂的推理任务。

### Memory（记忆）

为 Chain 和 Agent 提供上下文保存能力，支持多轮对话。

### Callbacks（回调）

在链执行各阶段插入自定义逻辑，常用于日志记录、监控和流式输出。

## 第一个 LangChain 应用

### 简单 LLM 调用

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage

# 初始化模型
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7,    # 创造性：0=确定性，1=随机性
    max_tokens=1000,
)

# 方式1：直接传入消息列表
messages = [
    SystemMessage(content="你是一个专业的 Python 开发助手。"),
    HumanMessage(content="用一句话解释什么是装饰器"),
]

response = llm.invoke(messages)
print(response.content)
# 输出：装饰器是一种用于修改函数或类行为的语法糖，通过包装函数实现功能增强。

# 方式2：直接传入字符串（适合简单场景）
response = llm.invoke("Python 中的 GIL 是什么？")
print(response.content)
```

### 流式输出

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini", streaming=True)

# 流式输出，逐字打印
for chunk in llm.stream("请写一首关于编程的短诗"):
    print(chunk.content, end="", flush=True)
print()  # 换行
```

### 使用 Ollama 本地模型

```python
from langchain_ollama import ChatOllama

# 使用本地 Ollama 模型（需要先安装并启动 Ollama）
llm = ChatOllama(
    model="qwen2.5:7b",
    temperature=0,
)

response = llm.invoke("你好，介绍一下自己")
print(response.content)
```

## PromptTemplate 使用

PromptTemplate 允许创建带有变量占位符的可复用提示词模板。

### 基础用法

```python
from langchain_core.prompts import PromptTemplate

# 字符串模板
template = PromptTemplate(
    input_variables=["product", "language"],
    template="请用{language}语言，为'{product}'写一段 50 字以内的产品描述。"
)

# 格式化模板（仅用于检查，实际使用时直接接入链）
prompt = template.format(product="智能咖啡机", language="中文")
print(prompt)
# 输出：请用中文语言，为'智能咖啡机'写一段 50 字以内的产品描述。
```

### ChatPromptTemplate（推荐）

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 构建对话提示词模板
chat_template = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的{role}，请用简洁专业的语言回答问题。"),
    ("human", "{question}"),
])

# LCEL 风格的链
llm = ChatOpenAI(model="gpt-4o-mini")
chain = chat_template | llm | StrOutputParser()

response = chain.invoke({
    "role": "数据科学家",
    "question": "解释一下过拟合问题"
})
print(response)
```

### Few-Shot 提示词模板

```python
from langchain_core.prompts import FewShotChatMessagePromptTemplate, ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# 定义示例
examples = [
    {"input": "快乐", "output": "悲伤"},
    {"input": "高大", "output": "矮小"},
    {"input": "炎热", "output": "寒冷"},
]

# 示例格式模板
example_template = ChatPromptTemplate.from_messages([
    ("human", "{input}"),
    ("ai", "{output}"),
])

# Few-shot 模板
few_shot_template = FewShotChatMessagePromptTemplate(
    example_prompt=example_template,
    examples=examples,
)

# 完整对话模板
final_template = ChatPromptTemplate.from_messages([
    ("system", "你是一个反义词专家，请给出单词的反义词。"),
    few_shot_template,
    ("human", "{word}"),
])

chain = final_template | ChatOpenAI(model="gpt-4o-mini") | StrOutputParser()
response = chain.invoke({"word": "勤奋"})
print(response)  # 输出：懒惰
```

## OutputParser（输出解析器）

OutputParser 将 LLM 的文本输出解析为结构化数据。

### StrOutputParser

```python
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# StrOutputParser：提取纯文本内容（去掉 AIMessage 包装）
chain = (
    ChatPromptTemplate.from_template("用一句话解释：{concept}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

result = chain.invoke({"concept": "递归"})
print(type(result))   # <class 'str'>
print(result)         # 递归是函数调用自身来解决问题的编程技术。
```

### JsonOutputParser

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 让模型输出 JSON 格式
parser = JsonOutputParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "请将用户信息提取为 JSON 格式，包含 name、age、city 字段。只返回 JSON，不要其他内容。"),
    ("human", "{text}"),
])

chain = prompt | ChatOpenAI(model="gpt-4o-mini") | parser

result = chain.invoke({
    "text": "我叫张三，今年 28 岁，住在上海。"
})
print(result)
# 输出：{'name': '张三', 'age': 28, 'city': '上海'}
print(type(result))   # <class 'dict'>
```

### PydanticOutputParser（结构化输出）

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field
from typing import List

# 定义输出结构
class CodeReview(BaseModel):
    score: int = Field(description="代码质量评分，1-10")
    issues: List[str] = Field(description="发现的问题列表")
    suggestions: List[str] = Field(description="改进建议列表")
    summary: str = Field(description="总结评价")

# 创建解析器
parser = PydanticOutputParser(pydantic_object=CodeReview)

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个代码审查专家。\n{format_instructions}"),
    ("human", "请审查以下代码：\n```python\n{code}\n```"),
])

chain = prompt | ChatOpenAI(model="gpt-4o-mini") | parser

result = chain.invoke({
    "code": "def add(a,b):\n    return a+b",
    "format_instructions": parser.get_format_instructions()
})

print(f"评分: {result.score}")
print(f"问题: {result.issues}")
print(f"建议: {result.suggestions}")
print(f"总结: {result.summary}")
```

## LangChain vs LCEL

### 传统 Chain 方式（旧版）

```python
# 旧版方式：使用 LLMChain 类（v0.1 风格，已逐渐废弃）
from langchain.chains import LLMChain
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

llm = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_template("翻译成英文：{text}")

chain = LLMChain(llm=llm, prompt=prompt)
result = chain.run(text="你好世界")  # 返回字符串
```

### LCEL 方式（推荐）

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
import asyncio

# LCEL：使用管道操作符 | 组合组件
chain = (
    ChatPromptTemplate.from_template("翻译成英文：{text}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

# 同步调用
result = chain.invoke({"text": "你好世界"})
print(result)

# 异步调用（适合 FastAPI 等异步框架）
async def async_call():
    result = await chain.ainvoke({"text": "你好世界"})
    return result

# 批量调用（并发处理多个输入）
results = chain.batch([
    {"text": "你好"},
    {"text": "再见"},
    {"text": "谢谢"},
])

# 流式输出
for chunk in chain.stream({"text": "请介绍人工智能的发展历史"}):
    print(chunk, end="", flush=True)
```

### LCEL 的优势对比

| 特性 | 旧版 Chain | LCEL |
|------|-----------|------|
| 语法 | 面向对象，较繁琐 | 管道操作符，简洁 |
| 流式支持 | 部分支持 | 原生支持 |
| 异步支持 | 有限 | 完整支持（ainvoke/astream） |
| 并行执行 | 手动处理 | RunnableParallel 原生支持 |
| 可观测性 | 一般 | 与 LangSmith 深度集成 |
| 类型提示 | 弱 | 强类型，IDE 友好 |

## LangSmith 调试

LangSmith 是 LangChain 官方提供的可观测性平台，可以追踪链的每一步执行，包括输入输出、耗时、Token 消耗等。

### 配置 LangSmith

```python
import os

# 方式1：在代码中设置（建议放在 .env 文件）
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "ls__your_api_key"
os.environ["LANGCHAIN_PROJECT"] = "my-ai-app"   # 项目名称，可自定义

# 之后所有的链调用都会自动上传到 LangSmith Dashboard
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

chain = (
    ChatPromptTemplate.from_template("解释：{concept}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

# 这次调用会被自动追踪，可在 https://smith.langchain.com 查看
result = chain.invoke({"concept": "向量数据库"})
```

### 控制台调试（无需注册）

```python
from langchain_core.tracers import ConsoleCallbackHandler
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

chain = (
    ChatPromptTemplate.from_template("总结：{text}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)

# 使用 ConsoleCallbackHandler 在控制台打印详细追踪信息
result = chain.invoke(
    {"text": "LangChain 是一个强大的 AI 开发框架"},
    config={"callbacks": [ConsoleCallbackHandler()]}
)
```

## 常见问题与最佳实践

### 问题1：API 调用频率限制（Rate Limit）

```python
from langchain_openai import ChatOpenAI

# max_retries 参数：遇到速率限制时自动重试
llm = ChatOpenAI(
    model="gpt-4o-mini",
    max_retries=3,           # 自动重试次数
    request_timeout=30,      # 超时时间（秒）
)
```

### 问题2：Token 数量控制

```python
from langchain_openai import ChatOpenAI
import tiktoken

# 生产环境配置
llm = ChatOpenAI(
    model="gpt-4o-mini",
    max_tokens=500,      # 最大输出 token 数
    temperature=0,       # 生产环境建议设为 0，确保输出稳定
)

# 预估输入 token 数，避免超出限制
def count_tokens(text: str, model: str = "gpt-4o-mini") -> int:
    encoder = tiktoken.encoding_for_model(model)
    return len(encoder.encode(text))

text = "这是一段需要计算 token 数量的文本"
print(f"Token 数量：{count_tokens(text)}")
```

### 问题3：多模型切换管理

```python
import os
from langchain_openai import ChatOpenAI
from langchain_ollama import ChatOllama
from langchain_core.language_models import BaseChatModel

def get_llm(use_local: bool = False) -> BaseChatModel:
    """根据环境配置返回对应的 LLM 实例"""
    if use_local:
        # 开发时用本地 Ollama 模型，节省 API 费用
        return ChatOllama(model="qwen2.5:7b")
    return ChatOpenAI(
        model=os.getenv("OPENAI_MODEL", "gpt-4o-mini"),
        temperature=float(os.getenv("LLM_TEMPERATURE", "0")),
    )

# 根据环境变量自动切换
llm = get_llm(use_local=os.getenv("ENV") == "development")
```

### 最佳实践总结

1. **始终使用 LCEL 而非旧版 Chain 类** - 更简洁，原生支持异步和流式
2. **生产环境将 temperature 设为 0** - 确保输出的稳定性和可重复性
3. **使用 PydanticOutputParser 处理结构化输出** - 避免手动解析字符串，类型安全
4. **用 LangSmith 追踪复杂链** - 便于排查问题和优化 Prompt
5. **将所有 Key 存入 .env 文件** - 避免硬编码，防止泄漏
6. **开发用本地模型（Ollama），生产用 API** - 节省开发成本
7. **为链添加输入验证** - 使用 Pydantic 校验输入参数，提前发现问题
