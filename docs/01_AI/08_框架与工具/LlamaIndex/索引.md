# LlamaIndex 索引

## 索引概述

LlamaIndex 的索引（Index）是整个框架的核心，它决定了文档如何被组织、存储和检索。不同的索引类型适合不同的使用场景，选择合适的索引对 RAG 系统的效果至关重要。

```
Document → Node Parser → Nodes → Embedding → Index
                                               ↓
                                        Query Engine
                                               ↓
                                         检索相关节点 → LLM → 答案
```

## 索引类型总览

| 索引类型 | 核心原理 | 适用场景 | 检索速度 |
|---------|---------|---------|---------|
| `VectorStoreIndex` | 向量相似度检索 | 语义问答，最常用 | 快 |
| `SummaryIndex` | 顺序遍历所有节点 | 需要综合全文信息 | 慢 |
| `KeywordTableIndex` | 关键词倒排索引 | 精确关键词匹配 | 快 |
| `KnowledgeGraphIndex` | 知识图谱三元组 | 实体关系查询 | 中 |
| `DocumentSummaryIndex` | 文档级别摘要 | 先定位文档再精细检索 | 中 |

## VectorStoreIndex（向量索引）

最常用的索引类型，通过计算问题与节点的向量相似度来检索相关内容。

### 基础构建

```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings
from llama_index.embeddings.openai import OpenAIEmbedding

Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# 方式一：从文档列表构建
documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(
    documents,
    show_progress=True,
)

# 方式二：从预处理好的 Node 列表构建
from llama_index.core.node_parser import SentenceSplitter

parser = SentenceSplitter(chunk_size=512, chunk_overlap=50)
nodes = parser.get_nodes_from_documents(documents)
index = VectorStoreIndex(nodes)
```

### 检索配置

```python
# 创建检索器（只做检索，不调用 LLM）
retriever = index.as_retriever(
    similarity_top_k=5,         # 返回最相似的 5 个节点
    similarity_cutoff=0.7,      # 相似度阈值，低于此值丢弃
)

# 检索相关节点
nodes = retriever.retrieve("什么是注意力机制？")
for node in nodes:
    print(f"[{node.score:.3f}] {node.text[:100]}")

# 创建查询引擎（检索 + LLM 生成答案）
query_engine = index.as_query_engine(
    similarity_top_k=3,
    response_mode="compact",
)
response = query_engine.query("请解释 Transformer 架构")
```

## SummaryIndex（摘要索引）

SummaryIndex 按顺序遍历所有节点，适合需要综合全文信息的场景（如：文章总结、全文分析）。

```python
from llama_index.core import SummaryIndex

# 构建摘要索引
index = SummaryIndex.from_documents(documents)

# 摘要索引适合全文问题
query_engine = index.as_query_engine(
    response_mode="tree_summarize",  # 树形递归汇总
)

# 适合"总结全文"类型的问题
response = query_engine.query("请对这份报告做一个完整的总结")
print(response.response)
```

## KeywordTableIndex（关键词索引）

基于关键词的倒排索引，适合精确匹配场景。

```python
from llama_index.core import KeywordTableIndex

# 构建关键词索引
index = KeywordTableIndex.from_documents(documents)

query_engine = index.as_query_engine(
    # 从问题中提取关键词进行匹配
    mode="simple",    # simple 或 rake
)

response = query_engine.query("Python 异步编程")
```

## KnowledgeGraphIndex（知识图谱索引）

将文档内容提取为三元组（主体-关系-客体），构建知识图谱。

```python
from llama_index.core import KnowledgeGraphIndex

# 构建知识图谱索引（需要 LLM 提取三元组，较慢）
index = KnowledgeGraphIndex.from_documents(
    documents,
    max_triplets_per_chunk=10,  # 每个节点最多提取 10 个三元组
    include_embeddings=True,     # 同时建立向量索引
)

# 可视化知识图谱
from pyvis.network import Network
g = index.get_networkx_graph()
net = Network(notebook=True, cdn_resources="in_line", directed=True)
net.from_nx(g)
net.show("knowledge_graph.html")

# 查询
query_engine = index.as_query_engine(
    include_text=True,
    response_mode="tree_summarize",
    embedding_mode="hybrid",  # 混合检索：关键词 + 向量
)
response = query_engine.query("LlamaIndex 和 LangChain 有什么关系？")
```

## 自定义节点解析（Node Parsers）

节点解析器负责将 Document 切分为 Node，是影响 RAG 效果的重要环节。

### SentenceSplitter（句子分割器）

```python
from llama_index.core.node_parser import SentenceSplitter

# 基于句子边界分割，避免在句子中间截断
splitter = SentenceSplitter(
    chunk_size=512,         # 每个 chunk 的最大 token 数
    chunk_overlap=50,       # 相邻 chunk 的重叠 token 数
    paragraph_separator="\n\n",   # 段落分隔符
    secondary_chunking_regex="[^,.;。？！]+[,.;。？！]?",  # 次级分割正则
)

nodes = splitter.get_nodes_from_documents(documents)
print(f"共生成 {len(nodes)} 个节点")
```

### SemanticSplitterNodeParser（语义分割器）

语义分割器通过计算相邻句子的语义相似度来确定切分点，避免在话题中间切断。

```python
from llama_index.core.node_parser import SemanticSplitterNodeParser
from llama_index.embeddings.openai import OpenAIEmbedding

embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# 语义分割：相似的句子放在同一个节点中
splitter = SemanticSplitterNodeParser(
    buffer_size=1,              # 相邻句子合并的缓冲大小
    breakpoint_percentile_threshold=95,  # 相似度低于此百分位才切分
    embed_model=embed_model,
)

nodes = splitter.get_nodes_from_documents(documents)
```

### HierarchicalNodeParser（层级分割器）

创建多粒度的层级节点，支持"小节点检索，大节点输入 LLM"的策略（Small-to-Big 检索）。

```python
from llama_index.core.node_parser import HierarchicalNodeParser, get_leaf_nodes
from llama_index.core.retrievers import AutoMergingRetriever
from llama_index.core import StorageContext

# 创建三个粒度层级：2048 → 512 → 128 tokens
node_parser = HierarchicalNodeParser.from_defaults(
    chunk_sizes=[2048, 512, 128]
)

nodes = node_parser.get_nodes_from_documents(documents)
leaf_nodes = get_leaf_nodes(nodes)  # 最细粒度的叶节点

# 构建索引（使用叶节点做检索）
storage_context = StorageContext.from_defaults()
storage_context.docstore.add_documents(nodes)  # 所有层级都存储

index = VectorStoreIndex(
    leaf_nodes,
    storage_context=storage_context,
)

# 使用 AutoMerging 检索器：检索小节点，返回其父节点
base_retriever = index.as_retriever(similarity_top_k=6)
retriever = AutoMergingRetriever(
    base_retriever,
    storage_context,
    verbose=True,
)

# 若检索到的叶节点超过父节点子节点数的 50%，则返回父节点
nodes = retriever.retrieve("问题")
```

## 增量索引更新

在生产环境中，文档经常需要更新，LlamaIndex 支持增量添加/删除文档。

```python
from llama_index.core import VectorStoreIndex, StorageContext
from llama_index.core import load_index_from_storage

# ---- 首次构建并保存 ----
index = VectorStoreIndex.from_documents(initial_documents)
index.storage_context.persist(persist_dir="./storage")

# ---- 后续加载并增量更新 ----
storage_context = StorageContext.from_defaults(persist_dir="./storage")
index = load_index_from_storage(storage_context)

# 插入新文档
new_doc = Document(text="新增的文档内容...", metadata={"source": "new_file.txt"})
index.insert(new_doc)

# 更新已有文档（先删除，再插入）
# 使用 doc_id 定位
old_doc_id = "existing-doc-id"
index.delete_ref_doc(old_doc_id, delete_from_docstore=True)
index.insert(updated_doc)

# 刷新文档（自动检测变化）
refreshed_docs = index.refresh_ref_docs(
    updated_documents,
    update_kwargs={"delete_kwargs": {"delete_from_docstore": True}}
)
print(f"刷新结果：{refreshed_docs}")  # [True, False, True] 表示哪些文档被更新

# 保存更新后的索引
index.storage_context.persist(persist_dir="./storage")
```

## 持久化存储

### 本地文件持久化

```python
from llama_index.core import StorageContext, load_index_from_storage

# 保存索引到本地
index.storage_context.persist(persist_dir="./my_index")

# 目录结构：
# ./my_index/
#   ├── docstore.json       # 文档存储
#   ├── index_store.json    # 索引元数据
#   ├── vector_store.json   # 向量数据（默认内存存储时）
#   └── graph_store.json    # 图存储（如果使用）

# 加载已保存的索引
storage_context = StorageContext.from_defaults(persist_dir="./my_index")
index = load_index_from_storage(storage_context)

# 如果有多个索引，指定 index_id
index = load_index_from_storage(storage_context, index_id="my_vector_index")
```

## 自定义向量存储后端

默认的向量存储在内存中，生产环境应使用持久化向量数据库。

### Chroma（轻量级本地向量库）

```python
# pip install llama-index-vector-stores-chroma chromadb
import chromadb
from llama_index.vector_stores.chroma import ChromaVectorStore
from llama_index.core import StorageContext, VectorStoreIndex

# 初始化 ChromaDB
chroma_client = chromadb.PersistentClient(path="./chroma_db")
chroma_collection = chroma_client.get_or_create_collection("my_documents")

# 创建向量存储
vector_store = ChromaVectorStore(chroma_collection=chroma_collection)
storage_context = StorageContext.from_defaults(vector_store=vector_store)

# 构建索引（向量自动保存到 ChromaDB）
index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
)

# 下次直接从 ChromaDB 加载（无需重新嵌入）
vector_store = ChromaVectorStore(chroma_collection=chroma_collection)
storage_context = StorageContext.from_defaults(vector_store=vector_store)
index = VectorStoreIndex.from_vector_store(
    vector_store,
    storage_context=storage_context,
)
```

### Qdrant（高性能向量数据库）

```python
# pip install llama-index-vector-stores-qdrant qdrant-client
from qdrant_client import QdrantClient
from llama_index.vector_stores.qdrant import QdrantVectorStore
from llama_index.core import StorageContext, VectorStoreIndex

# 连接 Qdrant（本地或远程）
client = QdrantClient(
    url="http://localhost:6333",  # 本地 Docker 实例
    # api_key="your-api-key",     # Qdrant Cloud 需要
)

# 创建向量存储
vector_store = QdrantVectorStore(
    client=client,
    collection_name="knowledge_base",
)

storage_context = StorageContext.from_defaults(vector_store=vector_store)

index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
    show_progress=True,
)

# 支持元数据过滤
from llama_index.core.vector_stores import MetadataFilter, MetadataFilters

filters = MetadataFilters(
    filters=[
        MetadataFilter(key="category", value="技术"),
        MetadataFilter(key="year", value="2024"),
    ]
)

query_engine = index.as_query_engine(
    similarity_top_k=5,
    filters=filters,  # 只在指定类别的文档中检索
)
```

### Pinecone（云端向量数据库）

```python
# pip install llama-index-vector-stores-pinecone pinecone-client
from pinecone import Pinecone, ServerlessSpec
from llama_index.vector_stores.pinecone import PineconeVectorStore
from llama_index.core import StorageContext, VectorStoreIndex

# 初始化 Pinecone
pc = Pinecone(api_key="your-pinecone-api-key")

# 创建或获取索引
index_name = "llamaindex-demo"
if index_name not in pc.list_indexes().names():
    pc.create_index(
        name=index_name,
        dimension=1536,   # text-embedding-3-small 的维度
        metric="cosine",
        spec=ServerlessSpec(cloud="aws", region="us-east-1"),
    )

pinecone_index = pc.Index(index_name)

# 创建 LlamaIndex 向量存储
vector_store = PineconeVectorStore(pinecone_index=pinecone_index)
storage_context = StorageContext.from_defaults(vector_store=vector_store)

index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
)
```

## 混合检索（Hybrid Search）

结合向量检索和关键词检索，提升召回效果：

```python
from llama_index.core.retrievers import QueryFusionRetriever
from llama_index.core import VectorStoreIndex, KeywordTableIndex

# 构建两种索引
vector_index = VectorStoreIndex.from_documents(documents)
keyword_index = KeywordTableIndex.from_documents(documents)

# 分别创建检索器
vector_retriever = vector_index.as_retriever(similarity_top_k=5)
keyword_retriever = keyword_index.as_retriever(similarity_top_k=5)

# 融合检索器：合并多路检索结果并去重排序
retriever = QueryFusionRetriever(
    retrievers=[vector_retriever, keyword_retriever],
    similarity_top_k=5,
    num_queries=3,          # 生成 3 个查询变体，提高召回
    mode="reciprocal_rerank",  # RRF 融合算法
    use_async=True,
    verbose=True,
)

nodes = retriever.retrieve("深度学习优化技巧")
```

## 最佳实践

### 选择合适的索引

```python
# 场景一：通用问答系统
# 使用 VectorStoreIndex，配合适当的 chunk_size
index = VectorStoreIndex.from_documents(docs)

# 场景二：需要综合全文的摘要问题
# 使用 DocumentSummaryIndex 先索引摘要，再细化检索
from llama_index.core import DocumentSummaryIndex
summary_index = DocumentSummaryIndex.from_documents(docs)

# 场景三：同时支持精确匹配和语义搜索
# 使用混合检索方案（向量 + 关键词融合）
```

### 调优建议

1. **chunk_size**：通常 256-512 适合精确问答，1024 适合需要上下文的总结
2. **similarity_top_k**：检索 3-5 个节点通常是最佳平衡点，太多会引入噪声
3. **chunk_overlap**：10%-20% 的重叠可以避免答案在边界被截断
4. **Embedding 模型**：中文场景建议使用 `BAAI/bge-large-zh-v1.5` 或 `text-embedding-3-large`
