# LlamaIndex RAG 评估

## 为什么需要评估 RAG

RAG 系统的质量受多个环节影响：文档切分策略、Embedding 模型选择、检索数量（top_k）、重排序逻辑、LLM 生成能力……任何一个环节出现问题都会导致答案质量下降。

**没有评估就没有改进。** 常见的 RAG 失败场景：

- **幻觉（Hallucination）**：LLM 生成了文档中根本不存在的信息
- **检索失败**：相关文档未被检索到，导致答案基于无关内容
- **答非所问**：检索到了相关文档，但 LLM 没有正确提炼答案
- **信息遗漏**：文档中有完整答案，但 LLM 只回答了部分

评估体系帮助我们量化上述问题，指导系统优化方向。

## 评估维度

### Faithfulness（忠实度）

衡量 LLM 的回答是否完全基于检索到的上下文，不包含外部知识或臆造内容。

- **高忠实度**：答案中的每个陈述都能在检索到的节点中找到依据
- **低忠实度**：答案包含文档中没有的信息（即幻觉）

```
忠实度 = 答案中有文档依据的陈述数 / 答案中的总陈述数
```

### Relevancy（相关性 / Answer Relevancy）

衡量 LLM 生成的答案是否真正回答了用户的问题。

- **高相关性**：答案直接回应了问题
- **低相关性**：答案偏题，或包含大量与问题无关的信息

### Context Precision（上下文精确度）

衡量检索到的节点中有多少是真正相关的（精确率视角）。

```
上下文精确度 = 相关节点数 / 检索到的总节点数
```

### Context Recall（上下文召回率）

衡量有多少相关信息被成功检索到（召回率视角），通常需要有标准答案才能计算。

```
上下文召回率 = 被检索到的相关节点数 / 所有相关节点数
```

## LlamaIndex 内置评估器

### FaithfulnessEvaluator

```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.core.evaluation import FaithfulnessEvaluator
from llama_index.llms.openai import OpenAI

# 准备索引和查询引擎
documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine()

# 初始化忠实度评估器（使用 LLM 作为裁判）
llm = OpenAI(model="gpt-4o", temperature=0)  # 评估用更强的模型
evaluator = FaithfulnessEvaluator(llm=llm)

# 执行查询
query = "Python 的 GIL 是什么？"
response = query_engine.query(query)

# 评估忠实度
eval_result = evaluator.evaluate_response(
    query=query,
    response=response,
)

print(f"是否忠实：{eval_result.passing}")      # True/False
print(f"评估分数：{eval_result.score}")        # 0.0 - 1.0
print(f"评估反馈：{eval_result.feedback}")     # LLM 的评估说明
```

### 批量评估

```python
import asyncio
from llama_index.core.evaluation import (
    FaithfulnessEvaluator,
    RelevancyEvaluator,
    BatchEvalRunner,
)

# 准备测试问题集
test_questions = [
    "什么是 Transformer 架构？",
    "注意力机制如何工作？",
    "BERT 和 GPT 有什么区别？",
    "如何微调预训练模型？",
]

# 批量获取响应
responses = [query_engine.query(q) for q in test_questions]

# 批量评估（异步并行，速度更快）
faithfulness_evaluator = FaithfulnessEvaluator(llm=llm)
relevancy_evaluator = RelevancyEvaluator(llm=llm)

runner = BatchEvalRunner(
    evaluators={
        "faithfulness": faithfulness_evaluator,
        "relevancy": relevancy_evaluator,
    },
    workers=4,  # 并行评估数量
)

eval_results = asyncio.run(
    runner.aevaluate_responses(
        queries=test_questions,
        responses=responses,
    )
)

# 汇总结果
import numpy as np

for metric_name, results in eval_results.items():
    scores = [r.score for r in results if r.score is not None]
    print(f"{metric_name}: 平均分 {np.mean(scores):.3f} ± {np.std(scores):.3f}")
```

## 评估数据集构建

好的评估需要高质量的测试数据集。LlamaIndex 提供了自动生成问答对的工具。

### 使用 LLM 自动生成问答对

```python
from llama_index.core.evaluation import generate_question_context_pairs
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.llms.openai import OpenAI

# 加载文档并切分为节点
documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents)

# 获取所有节点
nodes = list(index.docstore.docs.values())

# 使用 LLM 为每个节点生成问题
llm = OpenAI(model="gpt-4o-mini")

qa_dataset = generate_question_context_pairs(
    nodes=nodes[:50],     # 先用 50 个节点生成，避免费用过高
    llm=llm,
    num_questions_per_chunk=2,  # 每个节点生成 2 个问题
)

# 保存数据集
qa_dataset.save_json("./eval_dataset.json")
print(f"生成了 {len(qa_dataset.queries)} 个问答对")

# 查看示例
for qid, query in list(qa_dataset.queries.items())[:3]:
    relevant_docs = qa_dataset.relevant_docs[qid]
    print(f"问题：{query}")
    print(f"相关节点 ID：{relevant_docs}")
    print("---")
```

### 基础评估流程

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_recall,
    context_precision,
)
from ragas.integrations.llama_index import evaluate as ragas_evaluate
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

# 构建 RAG 系统
documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine(similarity_top_k=3)

# 准备评估数据集（问题 + 标准答案）
eval_questions = [
    "什么是 RAG？",
    "向量数据库的作用是什么？",
    "如何评估 LLM 的输出质量？",
]

eval_answers = [
    "RAG 是检索增强生成，结合了信息检索和语言模型生成能力。",
    "向量数据库用于存储和检索高维向量，支持相似度搜索。",
    "可以通过 BLEU、ROUGE 等自动指标或人工评估来评估 LLM 输出质量。",
]

# 执行 Ragas 评估（与 LlamaIndex 集成）
result = ragas_evaluate(
    query_engine=query_engine,
    metrics=[
        faithfulness,
        answer_relevancy,
        context_recall,
        context_precision,
    ],
    questions=eval_questions,
    ground_truths=eval_answers,  # context_recall 需要标准答案
)

# 打印评估报告
print(result)
# 输出类似：
# {'faithfulness': 0.923, 'answer_relevancy': 0.876,
#  'context_recall': 0.812, 'context_precision': 0.756}
```

## 评估结果分析与改进

### 常见问题诊断

```python
# 分析评估结果，找出改进方向
def diagnose_rag_issues(eval_df):
    """
    根据评估结果诊断 RAG 系统的问题
    """
    issues = []

    # 检查忠实度
    avg_faithfulness = eval_df['faithfulness'].mean()
    if avg_faithfulness < 0.8:
        issues.append({
            "问题": "忠实度低（可能存在幻觉）",
            "建议": [
                "在 Prompt 中加强指令：'只根据提供的文档回答，若文档中没有答案请说不知道'",
                "降低 LLM 的 temperature（接近 0）",
                "使用更强的 LLM（如 GPT-4o）",
            ]
        })

    # 检查上下文精确度
    avg_precision = eval_df.get('context_precision', pd.Series([1.0])).mean()
    if avg_precision < 0.7:
        issues.append({
            "问题": "上下文精确度低（检索到太多无关内容）",
            "建议": [
                "减小 similarity_top_k（从 5 减到 3）",
                "设置相似度阈值（similarity_cutoff=0.75）",
                "使用重排序模型（Reranker）过滤低相关节点",
            ]
        })

    # 检查上下文召回率
    avg_recall = eval_df.get('context_recall', pd.Series([1.0])).mean()
    if avg_recall < 0.7:
        issues.append({
            "问题": "上下文召回率低（漏掉了相关内容）",
            "建议": [
                "增大 similarity_top_k（从 3 增到 7）",
                "尝试混合检索（向量 + 关键词）",
                "使用查询扩展（生成多个查询变体）",
                "减小 chunk_size（256 或 128），更细粒度匹配",
            ]
        })

    # 检查答案相关性
    avg_relevancy = eval_df.get('answer_relevancy', pd.Series([1.0])).mean()
    if avg_relevancy < 0.8:
        issues.append({
            "问题": "答案相关性低（答非所问）",
            "建议": [
                "优化 System Prompt，明确回答格式要求",
                "使用 response_mode='compact' 或 'refine'",
                "检查是否有过多无关节点干扰 LLM",
            ]
        })

    return issues

# 使用示例
issues = diagnose_rag_issues(df)
for issue in issues:
    print(f"\n[问题] {issue['问题']}")
    for tip in issue['建议']:
        print(f"  - {tip}")
```

### A/B 对比不同配置

```python
import asyncio
import pandas as pd

async def evaluate_config(config_name, query_engine, qa_dataset):
    """评估单个配置"""
    runner = BatchEvalRunner(
        evaluators={
            "faithfulness": FaithfulnessEvaluator(llm=llm),
            "relevancy": RelevancyEvaluator(llm=llm),
        },
        workers=4,
    )

    queries = list(qa_dataset.queries.values())[:20]  # 取前 20 条评估
    responses = [query_engine.query(q) for q in queries]

    results = await runner.aevaluate_responses(
        queries=queries,
        responses=responses,
    )

    scores = {}
    for metric, res_list in results.items():
        valid_scores = [r.score for r in res_list if r.score is not None]
        scores[metric] = np.mean(valid_scores) if valid_scores else 0.0

    return {"config": config_name, **scores}


async def ab_test():
    # 配置 A：小 chunk，top_k=3
    from llama_index.core import Settings
    Settings.chunk_size = 256
    index_a = VectorStoreIndex.from_documents(documents)
    engine_a = index_a.as_query_engine(similarity_top_k=3)

    # 配置 B：大 chunk，top_k=5
    Settings.chunk_size = 512
    index_b = VectorStoreIndex.from_documents(documents)
    engine_b = index_b.as_query_engine(similarity_top_k=5)

    result_a = await evaluate_config("chunk256_top3", engine_a, qa_dataset)
    result_b = await evaluate_config("chunk512_top5", engine_b, qa_dataset)

    df = pd.DataFrame([result_a, result_b])
    print(df.to_string(index=False))

asyncio.run(ab_test())
```

## 最佳实践

1. **先建立 Baseline**：在优化前先评估当前系统的基准分数，确保每次改动都有对比依据

2. **分层评估**：分别评估检索质量（Retriever）和生成质量（Generator），明确瓶颈所在

3. **评估集要有代表性**：覆盖简单问题、复杂问题、需要多跳推理的问题、文档中没有答案的问题

4. **定期重新评估**：文档更新、模型升级后都要重新评估，防止性能退化

5. **人工审查低分样本**：自动评估不是万能的，定期人工抽查可以发现评估指标无法捕捉的问题
