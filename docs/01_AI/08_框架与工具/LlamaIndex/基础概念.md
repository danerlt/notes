# LlamaIndex 基础概念

## 简介

LlamaIndex（原名 GPT Index）是一个专注于 **数据索引与检索增强生成（RAG）** 的 Python 框架。与 LangChain 的大而全不同，LlamaIndex 的核心定位是：**让 LLM 高效地与外部数据交互**。

它解决了以下核心问题：
- 如何将大量文档高效加载并结构化
- 如何建立可检索的索引
- 如何让 LLM 基于索引内容回答问题

```
外部数据（PDF/网页/数据库）
        ↓  数据加载器
    Document（文档对象）
        ↓  Node Parser
     Node（文本节点）
        ↓  Embedding
      Index（向量索引）
        ↓  Query Engine
     LLM 回答问题
```

## 与 LangChain 的区别

| 维度 | LlamaIndex | LangChain |
|------|-----------|-----------|
| 核心定位 | 数据索引 + RAG | 通用 LLM 应用编排 |
| 学习曲线 | 相对平缓 | 概念较多，稍陡 |
| RAG 能力 | 极强，内置多种索引策略 | 需要手动组合组件 |
| Agent 支持 | 有，但非重点 | 非常完善 |
| 适用场景 | 知识库问答、文档检索 | 复杂 Agent、工作流编排 |

**选择建议**：
- 如果主要目标是构建文档问答、知识库 RAG 系统，优先选 LlamaIndex
- 如果需要复杂的 Agent、工具调用编排，选 LangChain 或两者结合

### 基础配置

```python
import os
from llama_index.core import Settings
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding

# 配置 API Key
os.environ["OPENAI_API_KEY"] = "your-api-key"

# 全局配置 LLM 和 Embedding 模型
Settings.llm = OpenAI(model="gpt-4o-mini", temperature=0.1)
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# 配置分块大小
Settings.chunk_size = 512       # 每个节点的 token 数量
Settings.chunk_overlap = 50     # 相邻节点的重叠 token 数
```

### 使用本地模型（Ollama）

```python
from llama_index.llms.ollama import Ollama
from llama_index.embeddings.huggingface import HuggingFaceEmbedding

# 使用 Ollama 运行本地 LLM
Settings.llm = Ollama(model="llama3.2", request_timeout=120.0)

# 使用 HuggingFace 本地 Embedding
Settings.embed_model = HuggingFaceEmbedding(
    model_name="BAAI/bge-small-zh-v1.5"  # 中文 embedding 模型
)
```

## 核心概念详解

### Document（文档对象）

`Document` 是 LlamaIndex 中的基本数据单元，代表一个完整的文档（如一个 PDF 文件、一个网页）。

```python
from llama_index.core import Document

# 手动创建 Document
doc = Document(
    text="LlamaIndex 是一个强大的 RAG 框架...",
    metadata={
        "source": "官方文档",
        "author": "LlamaIndex Team",
        "date": "2024-01-01"
    }
)

print(doc.doc_id)       # 自动生成的唯一 ID
print(doc.text[:100])   # 文档文本内容
print(doc.metadata)     # 元数据
```

### Node（文本节点）

`Node` 是 Document 切分后的最小处理单元。LLM 检索和处理的实际单位是 Node，而非整个 Document。

```python
from llama_index.core.schema import TextNode

# Node 包含文本片段及其位置信息
node = TextNode(
    text="这是文档的一个片段...",
    metadata={"page_number": 3, "source": "report.pdf"},
    # 记录在原始文档中的位置
    start_char_idx=100,
    end_char_idx=600,
)

# Node 之间可以建立关系
print(node.relationships)  # 父子关系、前后关系等
```

### Index（索引）

Index 是存储和组织 Node 的核心数据结构，支持多种检索方式。

```python
from llama_index.core import VectorStoreIndex

# 从文档列表构建索引
index = VectorStoreIndex.from_documents(documents)

# 索引内部会：
# 1. 将 Document 切分为 Node
# 2. 为每个 Node 计算 Embedding 向量
# 3. 将向量存入向量数据库（默认内存）
```

### Query Engine（查询引擎）

Query Engine 是执行问答的接口，它将用户问题转化为检索操作，再由 LLM 生成答案。

```python
# 从索引创建查询引擎
query_engine = index.as_query_engine(
    similarity_top_k=3,       # 检索最相关的 3 个节点
    response_mode="compact",  # 响应模式
)

# 执行查询
response = query_engine.query("LlamaIndex 有哪些核心功能？")
print(response.response)          # LLM 生成的答案
print(response.source_nodes)      # 引用的源节点列表
```

## 数据加载器

LlamaIndex 提供丰富的数据加载器，通过 `llama-hub` 可以访问 100+ 种数据源。

### SimpleDirectoryReader（本地文件）

```python
from llama_index.core import SimpleDirectoryReader

# 加载整个目录（自动识别 PDF、TXT、DOCX、MD 等格式）
documents = SimpleDirectoryReader(
    input_dir="./data",
    recursive=True,          # 递归处理子目录
    required_exts=[".pdf", ".txt", ".md"],  # 只处理指定格式
).load_data()

print(f"共加载 {len(documents)} 个文档")

# 加载单个文件
documents = SimpleDirectoryReader(
    input_files=["./data/report.pdf", "./data/manual.txt"]
).load_data()
```

### WebPageReader（网页内容）

```python
from llama_index.readers.web import SimpleWebPageReader

# 加载网页内容
loader = SimpleWebPageReader(html_to_text=True)  # 自动提取正文
documents = loader.load_data(
    urls=[
        "https://docs.llamaindex.ai/en/stable/",
        "https://python.langchain.com/docs/",
    ]
)
```

### 数据库加载器

```python
from llama_index.readers.database import DatabaseReader

# 从 SQL 数据库加载数据
reader = DatabaseReader(
    scheme="postgresql",
    host="localhost",
    port=5432,
    database="mydb",
    user="user",
    password="password",
)

# 将 SQL 查询结果作为文档
documents = reader.load_data(
    query="SELECT title, content FROM articles WHERE updated_at > '2024-01-01'"
)
```

### JSONReader（JSON 数据）

```python
from llama_index.readers.json import JSONReader

# 加载 JSON 文件
reader = JSONReader(
    levels_back=0,          # 从根节点开始处理
    clean_json=True,        # 清理格式
)
documents = reader.load_data("./data/products.json")
```

## 完整示例：构建文档问答系统

```python
import os
from llama_index.core import (
    VectorStoreIndex,
    SimpleDirectoryReader,
    Settings,
    StorageContext,
    load_index_from_storage,
)
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding

# ---- 1. 配置 ----
os.environ["OPENAI_API_KEY"] = "your-api-key"

Settings.llm = OpenAI(model="gpt-4o-mini")
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")
Settings.chunk_size = 512
Settings.chunk_overlap = 50

# ---- 2. 加载文档 ----
print("正在加载文档...")
documents = SimpleDirectoryReader("./docs").load_data()
print(f"共加载 {len(documents)} 个文档")

# ---- 3. 构建索引 ----
print("正在构建向量索引...")
index = VectorStoreIndex.from_documents(
    documents,
    show_progress=True,  # 显示进度条
)

# ---- 4. 持久化索引（避免重复构建）----
index.storage_context.persist(persist_dir="./storage")
print("索引已保存到 ./storage")

# ---- 5. 创建查询引擎 ----
query_engine = index.as_query_engine(
    similarity_top_k=5,
    response_mode="tree_summarize",  # 适合长文档的汇总模式
)

# ---- 6. 执行问答 ----
questions = [
    "这些文档的主要内容是什么？",
    "有哪些关键结论？",
    "请列出文档中提到的所有技术方案。",
]

for question in questions:
    print(f"\n问题：{question}")
    response = query_engine.query(question)
    print(f"答案：{response.response}")
    print(f"引用了 {len(response.source_nodes)} 个源节点")
```

## 响应模式（Response Mode）

Query Engine 支持多种响应模式，适用于不同场景：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `compact` | 将多个节点合并后一次性输入 LLM | 默认，平衡速度与质量 |
| `refine` | 逐个节点迭代优化答案 | 需要高质量、综合多源的答案 |
| `tree_summarize` | 树形递归汇总 | 大量节点时性能更好 |
| `simple_summarize` | 简单拼接后汇总 | 快速摘要 |
| `no_text` | 只返回检索到的节点，不调用 LLM | 纯检索场景 |

```python
# 选择不同响应模式
query_engine = index.as_query_engine(
    response_mode="refine",  # 迭代精化，质量更高但速度慢
    similarity_top_k=5,
)
```

## 常见问题

### Q：文档加载后乱码怎么办？

```python
# 指定编码
documents = SimpleDirectoryReader(
    "./data",
    encoding="utf-8",  # 或 "gbk" 处理中文 Windows 文件
).load_data()
```

### Q：如何控制分块策略？

```python
from llama_index.core.node_parser import SentenceSplitter

# 自定义节点解析器
parser = SentenceSplitter(
    chunk_size=256,       # 更小的 chunk 提高精确度
    chunk_overlap=20,
    paragraph_separator="\n\n",  # 按段落分割
)
Settings.text_splitter = parser
```

### Q：如何查看检索到的源文档？

```python
response = query_engine.query("你的问题")

# 查看每个引用节点的详情
for i, node in enumerate(response.source_nodes):
    print(f"\n--- 源节点 {i+1} ---")
    print(f"相似度分数：{node.score:.4f}")
    print(f"来源文件：{node.metadata.get('file_name', '未知')}")
    print(f"内容预览：{node.text[:200]}...")
```
