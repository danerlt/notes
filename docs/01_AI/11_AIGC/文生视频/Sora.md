# Sora

## 简介

Sora 是 OpenAI 于 2024 年 2 月发布的文生视频模型，以其惊人的视频质量和长达 60 秒的生成能力震惊了整个 AI 领域。Sora 不仅能够根据文字描述生成高质量视频，还能扩展已有视频、填充缺失帧、以及根据图片生成视频。

OpenAI 将 Sora 定位为"世界模拟器"（World Simulator），认为它学到了物理世界的运动规律，而不仅仅是视觉模式。

**关键特性：**
- 最长 60 秒的高质量视频生成
- 支持多种分辨率和宽高比（最高 1080p）
- 准确理解自然语言提示
- 多镜头场景与角色一致性
- 视频扩展、填充与混合能力

## 技术原理

### Video Diffusion Transformer

Sora 的核心架构是 **Video Diffusion Transformer（DiT）**，将扩散模型与 Transformer 结合。

**与传统视频扩散模型的区别：**

| 维度 | 传统 UNet 路线 | Sora DiT 路线 |
|------|---------------|--------------|
| 骨干网络 | UNet（CNN 为主） | Transformer |
| 可扩展性 | 有限 | 可大规模扩展 |
| 全局建模 | 局部感受野为主 | 全局注意力 |
| 灵活分辨率 | 固定尺寸 | 原生多分辨率 |

**整体流程：**

```
文本输入
    ↓ CLIP / T5 文本编码器
文本条件向量
    ↓
视频潜变量（Video Latent）
+ 加入高斯噪声
    ↓
Video DiT（多层 Transformer）
    ↓ 去噪迭代
干净的视频潜变量
    ↓ 视频 VAE Decoder
最终视频
```

### Spacetime Patches（时空补丁）

这是 Sora 最核心的创新之一。

**传统方法的问题：**
- 处理视频时通常将时间和空间独立处理
- 或者将帧展平为 2D 序列，丢失时序关联

**Spacetime Patches 的思路：**

1. **视频压缩**：首先用视频 VAE 将原始视频压缩到低维潜变量空间
2. **时空切块**：将压缩后的视频潜变量切分为固定大小的 3D 块（宽 × 高 × 时间）
3. **展平为序列**：将这些 3D 块展平，作为 Transformer 的输入 token
4. **统一建模**：Transformer 在时间和空间维度上同时建模注意力

```
视频帧序列（T × H × W × C）
    ↓ 视频 VAE 压缩
潜变量（t × h × w × c）
    ↓ Patchify（切成 3D 块）
Spacetime Tokens 序列
    ↓ Transformer 处理
输出 Tokens
    ↓ Unpatchify + VAE Decode
重建视频
```

**优势：**
- 天然支持不同时长、分辨率的视频（token 数量不同，但结构一致）
- 时空信息统一建模，避免了独立处理时的信息割裂
- 规模可扩展：更大模型 = 更多参数处理同样的 token 序列

### 训练数据与规模

Sora 的训练数据规模极大，包含：
- 互联网上的大量视频数据
- 对视频进行自动字幕标注（利用 GPT-4V）
- 视频时长从几秒到几分钟不等
- 多种分辨率、宽高比的数据

通过在不同分辨率和时长的原生数据上训练，Sora 获得了灵活适应各种输入规格的能力。

## 核心能力

### 文生视频

根据文字描述直接生成视频，支持丰富的场景、动作和情感描述。

**能力亮点：**
- 复杂场景构建（多角色、复杂背景）
- 流畅的摄像机运动
- 对物理现象的准确模拟（如水、光线）
- 情感和氛围的传递

### 图生视频

以一张图片为起点，生成向前或向后延伸的视频。

### 视频扩展

将已有的短视频向前或向后扩展，保持内容连贯性。

### 视频混合（Video-to-Video）

将两段视频逐渐融合/过渡。

### 连接与填充

给定视频的开头和结尾，自动生成中间过渡帧。

## 局限性

尽管 Sora 非常强大，仍存在已知局限：

### 物理模拟缺陷

- **玻璃破碎**：碎片行为不符合物理规律
- **食物咀嚼**：人物进食动作不自然
- **复杂流体**：水流、火焰在极端情况下失真
- **因果关系**：有时物体"凭空出现"或"消失"

### 空间关系混淆

- 左右、上下方向有时错误
- 多个物体的相对位置关系可能违背描述

### 角色一致性

- 长视频中角色外貌可能发生细微变化
- 多角色场景中人物可能相互"融合"

### 时间逻辑

- 对于需要时间顺序理解的场景（如"先放糖后加水"）处理不准确

## 提示词技巧

### 基本结构

```
[主体描述] [动作/状态] [场景/背景] [风格] [摄像机] [光线]
```

### 有效提示词要素

**主体描述（具体化）：**
```
# 模糊
一只狗在跑

# 具体
一只金色的拉布拉多猎犬在绿色草地上奔跑，尾巴高高扬起，
表情愉悦，毛发随风飘动
```

**摄像机运动（专业术语效果更好）：**
```
- 推进镜头（dolly in / push in）
- 拉远镜头（pull out / dolly out）
- 环绕跟拍（arc shot）
- 俯视航拍（bird's eye view / aerial shot）
- 手持晃动（handheld）
- 固定镜头（static camera）
- 慢速平移（slow pan）
```

**风格词汇：**
```
- 电影感：cinematic, film grain, anamorphic lens
- 纪录片：documentary style, naturalistic
- 广告风：commercial, vibrant colors, high production value
- 动画感：animated, stylized, cartoon
- 赛博朋克：cyberpunk, neon lights, dystopian
```

**光线描述：**
```
- 黄金时段：golden hour, warm sunlight
- 蓝调时刻：blue hour, twilight
- 工作室光：studio lighting, three-point lighting
- 自然漫射：overcast, soft diffused light
- 戏剧性：dramatic lighting, chiaroscuro
```

### 实用示例

**场景一：产品广告风格**
```
一瓶高档香水放置在大理石台面上，香气化作金色粒子向外扩散，
慢镜头，奢华风格，黑色背景，强调灯打亮瓶身，
电影感镜头，4K 质感
```

**场景二：自然风光纪录片**
```
延时摄影：日出时分，西藏高原上云海在山峰间流动，
金色阳光逐渐照亮雪峰，
航拍视角缓慢向前推进，
宏大壮观，自然纪录片风格
```

**场景三：角色动作场景**
```
一位身着红色连衣裙的女性在雨中的巴黎街道行走，
街灯反射在湿润的石板路上，
侧面跟拍镜头，慢动作，
写实电影风格，香颂音乐氛围感
```

### 提示词优化技巧

1. **负向提示**：Sora 目前不支持负向提示词，需要通过正向描述规避不想要的内容
2. **迭代生成**：先生成简单版本，根据结果调整提示词
3. **分段描述**：对于复杂场景，按照时间顺序描述场景变化
4. **参考风格**：引用已知的电影、摄影师或艺术风格

**选择 Sora 的场景：**
- 追求最高质量和物理真实感

- 有 ChatGPT Pro 订阅

## 最佳实践

### 创作工作流

```
1. 概念定稿
   确定视频主题、风格参考、时长需求

2. 提示词草稿
   写出基本描述，包含主体、动作、场景

3. 风格化
   添加摄像机语言、光线描述、参考风格

4. 首次生成
   使用简洁版提示词生成第一版

5. 迭代优化
   根据结果调整提示词细节

6. 后期处理
   用剪辑软件拼接、配音、调色
```

### 常见问题解决

**问题：角色手部变形**
- 减少对手部的直接描述
- 避免要求手持物品的特写镜头

**问题：文字/Logo 无法正确生成**
- Sora 对文字生成能力有限，建议后期叠加

**问题：多角色场景混乱**
- 减少同帧出现的角色数量
- 用颜色、服装明确区分不同角色

**问题：长视频后半段质量下降**
- 使用视频扩展功能分段生成
- 在关键节点重新生成
