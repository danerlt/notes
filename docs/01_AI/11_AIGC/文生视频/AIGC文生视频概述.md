# AIGC 文生视频概述

## 什么是文生视频

文生视频（Text-to-Video，T2V）是 AIGC（AI Generated Content）领域的重要分支，指通过自然语言描述自动生成视频内容的技术。用户只需输入一段文字提示词，AI 模型即可生成对应的视频片段。

这一技术融合了计算机视觉、自然语言处理、生成模型等多个领域的最新进展，代表了 AI 内容生成能力的重大突破。

---

## 发展历程

### 早期探索阶段（2022 年之前）

- **GAN 路线**：早期文生视频主要基于 GAN（生成对抗网络），生成质量低、分辨率有限，视频长度通常只有几帧
- **VideoGAN、MoCoGAN** 等工作奠定了视频生成的基础框架
- 主要局限：训练不稳定、多样性差、无法接受文字控制

### 扩散模型崛起（2022-2023）

- **Meta Make-A-Video（2022）**：首批基于扩散模型的文生视频系统，通过图像扩散模型扩展到视频
- **Google Imagen Video（2022）**：级联扩散模型生成高清视频
- **ModelScope（2023）**：阿里达摩院开源的文生视频模型，推动了社区发展
- **Stable Video Diffusion（2023）**：Stability AI 发布，开源生态进一步繁荣
- **Runway Gen-2（2023）**：商业化产品走向实用

### 大模型涌现（2024-2025）

- **Sora（2024.02）**：OpenAI 发布，震惊业界，首次实现长达 60 秒的高质量视频生成
- **Kling 可灵（2024.06）**：快手发布，国产视频生成的代表性产品
- **HunyuanVideo（2024.12）**：腾讯开源，开源视频生成质量新标杆
- **Runway Gen-3 Alpha（2024）**：Runway 迭代升级，画质与一致性大幅提升
- **Pika 2.0（2024）**：专注短视频生成，交互性强
- **Wan2.1（2025）**：阿里万象开源，多任务视频生成模型

## 主流技术路线

### Diffusion-based（扩散模型）

目前最主流的技术路线，核心思想是从噪声中逐步去噪生成目标视频。

**原理流程：**

```
文字提示词
    ↓ Text Encoder（如 CLIP、T5）
文本特征向量
    ↓ 与噪声视频潜变量融合
扩散去噪过程（UNet 或 Transformer）
    ↓ 反复迭代（通常 20-50 步）
视频潜变量
    ↓ VAE Decoder
最终视频帧
```

**代表模型：**
- Stable Video Diffusion（SVD）
- HunyuanVideo
- CogVideoX
- Wan2.1

**优点：**
- 生成质量高，细节丰富
- 支持灵活的条件控制
- 开源生态完善

**缺点：**
- 推理速度慢（需要多步去噪）
- 计算资源消耗大

### Autoregressive（自回归）

类似语言模型的方式，将视频 token 化后逐步预测下一帧或下一个 token。

**代表模型：**
- VideoGPT
- MAGVIT-v2（Google）
- 部分 Sora 架构推测也包含自回归成分

**优点：**
- 天然支持长视频生成
- 与语言模型框架统一，便于多模态扩展

**缺点：**
- 生成速度更慢（逐 token 生成）
- 累积误差导致长视频质量下降

### Video Diffusion Transformer（DiT 路线）

结合 Diffusion 与 Transformer 架构，是当前最前沿的路线。

- 使用 Transformer 替代传统 UNet 作为去噪网络
- 通过 Spacetime Patches 统一处理时空信息
- 代表：Sora、HunyuanVideo、CogVideoX-5B

- 专注于简单易用的视频生成体验
- 支持局部修改（Pikaffects）
- 社交媒体风格内容生成效果好

### Kling 可灵

- 快手大模型团队出品
- 国产视频生成领域标杆
- 支持超长视频（最长 3 分钟）

- 社区围绕 ComfyUI 构建了丰富工作流

## 技术挑战

### 时序一致性

视频生成中最核心的挑战之一，要求生成的视频帧之间保持连贯性。

**问题表现：**
- 人物/物体在帧间突然变形
- 背景元素随机消失或替换
- 运动轨迹不自然（如手臂穿透身体）

**应对技术：**
- 时序注意力机制（Temporal Attention）：在相邻帧之间建立注意力连接
- 光流约束：引入光流预测损失，保证像素运动的平滑性
- 视频 VAE：使用专门为视频设计的变分自编码器，在压缩域保持时序信息

### 物理合理性

生成的视频需要符合现实世界的物理规律。

**常见问题：**
- 水、火、烟等流体动力学不真实
- 刚体碰撞效果违反物理定律
- 人物运动违背重力规律
- 光影变化不自然

**研究方向：**
- 引入物理仿真先验
- 构建包含物理标注的训练数据
- Sora 的 "world model" 理念：通过海量视频学习物理规律

### 长视频生成

生成超过 10 秒的视频时质量急剧下降。

**挑战原因：**
- 计算复杂度随时长二次方增长
- 累积误差导致后段视频质量下降
- 语义连贯性难以在长时间维持

**解决思路：**
- 滑动窗口推理
- 分段生成后拼接
- 层次化生成（先生成关键帧，再插值）

### 可控性

用户对生成内容的精细控制仍然是难题。

**控制维度：**
- 摄像机运动（平移、旋转、缩放）
- 人物动作与表情
- 场景风格与氛围
- 多角色互动

## 评估指标

### 视觉质量

| 指标 | 说明 |
|------|------|
| **FVD（Fréchet Video Distance）** | 类似 FID，衡量生成视频与真实视频的分布差异，越低越好 |
| **PSNR** | 峰值信噪比，衡量像素级重建质量 |
| **SSIM** | 结构相似性，感知质量评估 |
| **LPIPS** | 学习型感知图像块相似度 |

### 文本对齐

| 指标 | 说明 |
|------|------|
| **CLIP Score** | 文本与视频帧的 CLIP 特征相似度 |
| **BLIPScore** | 使用 BLIP 模型评估语义对齐 |
| **Human Evaluation** | 人工评估，通常是黄金标准 |

### 时序一致性

| 指标 | 说明 |
|------|------|
| **Temporal Consistency** | 相邻帧特征相似度 |
| **Warping Error** | 光流扭曲误差，衡量运动平滑性 |
| **VideoScore** | 综合评估视频质量的神经网络指标 |

### EvalCrafter 评测框架

目前业界较权威的文生视频综合评测框架，包含：
- 视觉质量维度
- 动作效果维度
- 文本对齐维度
- 物理规律合理性维度

## 应用场景

### 广告与营销

- **产品视频**：根据产品图片和描述自动生成广告素材
- **创意脚本**：将文案快速转化为视觉化演示
- **A/B 测试素材**：批量生成多版本视频进行效果测试

### 影视与娱乐

- **分镜预演（Previs）**：导演用文生视频快速验证场景概念
- **背景板制作**：生成影视后期所需的背景环境视频
- **概念展示**：游戏开发团队快速展示场景和角色概念

### 教育与培训

- **知识可视化**：将抽象知识转化为直观视频
- **教学案例**：生成历史场景、科学现象演示视频
- **多语言内容**：一次生成，多语言配音

### 社交媒体创作

- **短视频内容**：博主快速生成 Reel、TikTok 内容
- **Meme 和趣味视频**：创意短视频自动化生成
- **虚拟形象**：数字人视频制作

### 企业应用

- **培训视频**：企业内部培训材料自动生成
- **产品说明**：操作指南视频化
- **数据可视化**：将业务数据转化为动态图表视频

## 使用建议与最佳实践

### 提示词编写原则

好的文生视频提示词结构：`[场景描述] + [主体动作] + [视觉风格] + [摄像机运动] + [光线氛围]`

示例：
```
一只金色的猫咪在阳光明媚的草地上奔跑，
慢镜头，电影感，浅景深，自然光，
镜头跟随移动
```

### 工具选择建议

- **追求最高质量**：Sora（付费订阅）
- **国内访问便利**：Kling 可灵
- **开源自部署**：HunyuanVideo + ComfyUI
- **简单易上手**：Pika
- **专业创作**：Runway Gen-3

### 工作流集成

1. 用文生图（Midjourney/FLUX）确定视觉风格
2. 使用图生视频功能保持风格一致性
3. 多段视频生成后用剪映/PR 拼接
4. 用 AI 配音和背景音乐完善成品
