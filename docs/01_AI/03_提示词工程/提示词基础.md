# 提示词基础

## 概念介绍

提示词工程（Prompt Engineering）是通过设计和优化输入提示词，引导大语言模型产生更准确、更有用输出的技术。好的提示词能让同一个模型产生截然不同的结果——这是使用 LLM 最重要的技能之一。

提示词工程的重要性：
- **无需微调**：通过提示词即可适配各类任务
- **即时生效**：修改提示词立即看到效果
- **通用性强**：适用于所有主流 LLM
- **成本低**：相比微调，成本极低

## 核心原理

### 提示词的基本结构

一个完整的提示词通常包含以下要素：

```
[角色设定]     → 你是谁，有什么专业背景
[任务描述]     → 你需要做什么
[上下文信息]   → 相关背景信息
[输入数据]     → 需要处理的具体内容
[输出格式]     → 期望的输出形式
[约束条件]     → 限制和注意事项
[示例（可选）] → 期望输出的示例
```

不是每个提示词都需要包含全部要素，根据任务复杂度选择组合。

### LLM 的工作机制对提示词的影响

理解 LLM 的特性有助于设计更好的提示词：
- **Token 预测**：模型逐 Token 生成，前文影响后文
- **注意力机制**：重要信息应放在开头或结尾
- **上下文窗口**：超过窗口的信息会被"遗忘"
- **对话历史**：多轮对话中，历史信息影响当前回复

## 提示词设计原则

### 1. 清晰具体（Be Specific）

模糊的指令会产生不确定的输出：

**差的提示词**：
```
帮我写一篇关于 AI 的文章
```

**好的提示词**：
```
请写一篇 500 字左右的文章，面向没有技术背景的读者，
介绍人工智能在医疗诊断领域的应用。
要求：使用日常语言，避免专业术语，包含 2-3 个具体例子。
```

### 2. 角色设定（Role Prompting）

赋予模型明确角色能提升专业度：

```
你是一位有 10 年经验的 Python 高级工程师，
精通性能优化和代码审查。请...
```

### 3. 结构化输出

明确要求输出格式，便于后续处理：

```
请分析以下产品评论，并以 JSON 格式输出结果：
{
  "sentiment": "正面/负面/中性",
  "score": 1-5,
  "key_points": ["要点1", "要点2"],
  "summary": "一句话总结"
}
```

### 4. 提供示例（Few-Shot）

给出期望的输入输出示例，效果通常优于纯描述。

### 5. 分步骤思考

对于复杂任务，要求模型分步推理：

```
请一步一步思考以下问题：...
```

## 代码示例（Python）

### 基础提示词调用

```python
from openai import OpenAI

client = OpenAI()

def call_llm(
    prompt: str,
    system: str = "你是一个有帮助的 AI 助手。",
    model: str = "gpt-4o",
    temperature: float = 0.7
) -> str:
    """基础 LLM 调用封装"""
    response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system},
            {"role": "user", "content": prompt}
        ],
        temperature=temperature
    )
    return response.choices[0].message.content


# 示例：文本分类
text = "这款手机的电池续航太差了，充一次电只能用半天，非常失望。"

prompt = f"""请对以下产品评论进行情感分析。

评论内容：
{text}

请输出以下 JSON 格式：
{{
  "sentiment": "正面/负面/中性",
  "confidence": 0-1之间的浮点数,
  "reason": "判断理由（一句话）"
}}

只输出 JSON，不要有其他内容。"""

result = call_llm(prompt)
print(result)
```

### 结构化输出提示词

```python
from pydantic import BaseModel
from typing import List
import json

class ProductAnalysis(BaseModel):
    product_name: str
    sentiment: str
    score: int
    pros: List[str]
    cons: List[str]
    recommendation: str

def analyze_product_review(review: str) -> ProductAnalysis:
    """分析产品评论，返回结构化数据"""

    prompt = f"""请分析以下产品评论，提取关键信息。

评论：
{review}

请严格按照以下 JSON 格式输出，不要有额外内容：
{{
  "product_name": "产品名称（从评论中提取，不明确则填'未知'）",
  "sentiment": "正面/负面/中性",
  "score": 1-5的整数评分,
  "pros": ["优点1", "优点2"],
  "cons": ["缺点1", "缺点2"],
  "recommendation": "是否推荐购买，原因是什么（一句话）"
}}"""

    result = call_llm(prompt, temperature=0.1)  # 低温度提高输出稳定性

    # 解析 JSON
    try:
        data = json.loads(result)
        return ProductAnalysis(**data)
    except (json.JSONDecodeError, ValueError) as e:
        # 容错处理：提取 JSON 块
        import re
        json_match = re.search(r'\{.*\}', result, re.DOTALL)
        if json_match:
            data = json.loads(json_match.group())
            return ProductAnalysis(**data)
        raise ValueError(f"无法解析输出: {result}") from e


review = "买了这款 AirPods Pro 2，音质和降噪效果非常好，佩戴也很舒适。就是价格有点贵，而且续航不够长。总体来说还是值得购买的。"
analysis = analyze_product_review(review)
print(f"情感: {analysis.sentiment}")
print(f"评分: {analysis.score}/5")
print(f"优点: {analysis.pros}")
print(f"缺点: {analysis.cons}")
```

### 提示词模板管理

```python
from string import Template

class PromptTemplate:
    """简单的提示词模板管理器"""

    TEMPLATES = {
        "summarize": Template("""请用$max_words个字以内总结以下$content_type：

$content

要求：
- 保留最重要的信息
- 使用简洁的语言
- 不要添加原文没有的内容"""),

        "translate": Template("""请将以下$source_lang文本翻译成$target_lang：

$text

要求：
- 保持原文的语气和风格
- 专业术语保持原文并在括号内注明$target_lang
- 不要添加额外解释"""),

        "extract_info": Template("""从以下文本中提取$info_type信息：

$text

请以 JSON 格式输出提取的信息。"""),
    }

    @classmethod
    def render(cls, template_name: str, **kwargs) -> str:
        template = cls.TEMPLATES.get(template_name)
        if not template:
            raise ValueError(f"模板不存在: {template_name}")
        return template.substitute(**kwargs)


# 使用示例
prompt = PromptTemplate.render(
    "summarize",
    max_words=100,
    content_type="技术文档",
    content="Transformer 是一种基于注意力机制的神经网络架构..." * 5
)
result = call_llm(prompt)
print(result)
```

## 常见用法

### 提示词调试技巧

```python
def test_prompt_variants(base_prompt: str, variants: list, test_input: str):
    """测试不同提示词变体的效果"""
    results = []

    for i, variant in enumerate(variants):
        prompt = variant.format(input=test_input)
        response = call_llm(prompt, temperature=0.1)  # 低温度确保可重复性
        results.append({
            "variant_id": i + 1,
            "prompt": variant[:100] + "...",
            "response": response
        })

    return results

# 对比不同指令风格
variants = [
    "总结以下文本：\n{input}",
    "请用3句话总结以下文本的核心内容：\n{input}",
    "作为一名编辑，请提炼以下文本的关键信息（3个要点）：\n{input}",
]

test_input = "人工智能正在改变我们的工作和生活方式..."
results = test_prompt_variants("", variants, test_input)
for r in results:
    print(f"\n变体 {r['variant_id']}:")
    print(f"响应: {r['response']}")
```

## 小结

- **清晰具体**是提示词设计的第一原则，模糊的指令导致不稳定的输出
- **角色设定**能显著提升专业领域任务的质量
- **结构化输出**（要求 JSON/Markdown 格式）便于程序后续处理
- **低温度（0.0-0.3）** 用于需要确定性输出的任务（分类、提取），**高温度（0.7-1.0）** 用于创意任务
- 提示词设计是迭代过程：先写初版 → 测试 → 分析失败案例 → 改进 → 循环
- 2024-2025 年最佳实践：使用 **Structured Outputs**（OpenAI）或 **Instructor 库** 确保模型输出符合预定义 Schema
