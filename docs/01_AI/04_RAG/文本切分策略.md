# 文本切分策略

## 概念介绍

文本切分（Text Chunking / Text Splitting）是 RAG 系统中至关重要的预处理步骤。由于向量数据库和 LLM 上下文窗口的限制，不能将整个文档直接处理，需要将文档拆分为合适大小的片段（Chunk）。

切分策略直接影响：
- **检索精度**：Chunk 太大，相关信息淹没在无关内容中；太小，语义完整性差
- **存储成本**：Chunk 越多，存储和索引成本越高
- **生成质量**：检索到的 Chunk 是否包含完整的上下文信息

核心参数：
- **chunk_size**：每个 Chunk 的最大字符/Token 数
- **chunk_overlap**：相邻 Chunk 之间的重叠字符数（避免切断语义单元）

## 核心原理

### 常见切分策略对比

| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 固定大小切分 | 按固定字符数切分 | 简单，均匀 | 可能切断语义单元 | 通用文本 |
| 递归字符切分 | 按分隔符递归切分 | 尽量保留语义完整 | 参数调优复杂 | 大多数场景 |
| 按文档结构切分 | 按标题/段落切分 | 保留文档结构 | 依赖文档格式规范 | 结构化文档 |
| 语义切分 | 按语义边界切分 | 语义完整性最高 | 计算成本高 | 高质量要求场景 |
| 代码切分 | 按语法单元切分 | 保留代码逻辑 | 需要语言解析器 | 代码文件 |

### Chunk 大小选择指南

```
chunk_size 选择建议：
- 100-200 字：FAQ、简短问答、标题式文档
- 300-500 字：通用文档（最常用范围）
- 500-800 字：详细说明文档、学术论文
- 800-1200 字：需要完整上下文的法律/技术文档

overlap 建议：
- 通常设为 chunk_size 的 10%-20%
- chunk_size=500 时，overlap=50-100 是合理范围
- overlap 过大：信息重复，浪费存储；过小：边界语义丢失
```

### 方法1：固定大小切分（CharacterTextSplitter）

```python
# 最简单的切分方式：按字符数固定切分
char_splitter = CharacterTextSplitter(
    separator="\n",      # 切分分隔符
    chunk_size=200,      # 每个 chunk 最大字符数
    chunk_overlap=20,    # 相邻 chunk 的重叠字符数
    length_function=len, # 长度计算函数
)

chunks = char_splitter.split_text(sample_text)
print(f"固定大小切分：{len(chunks)} 个 chunk")
for i, chunk in enumerate(chunks, 1):
    print(f"Chunk {i} ({len(chunk)} 字):\n{chunk[:100]}...\n")
```

### 方法2：递归字符切分（推荐，最常用）

```python
# RecursiveCharacterTextSplitter 是最常用的切分器
# 按优先级尝试不同分隔符：["\n\n", "\n", "。", "！", "？", " ", ""]
recursive_splitter = RecursiveCharacterTextSplitter(
    chunk_size=400,
    chunk_overlap=50,
    length_function=len,
    # 中文优化的分隔符列表
    separators=[
        "\n\n",     # 段落分隔（最高优先级）
        "\n",       # 换行
        "。",       # 中文句号
        "！",       # 中文感叹号
        "？",       # 中文问号
        "；",       # 中文分号
        "，",       # 中文逗号
        " ",        # 空格
        "",         # 字符级别（最低优先级）
    ],
    is_separator_regex=False
)

chunks = recursive_splitter.split_text(sample_text)
print(f"递归字符切分：{len(chunks)} 个 chunk")
for i, chunk in enumerate(chunks, 1):
    print(f"\nChunk {i} ({len(chunk)} 字):\n{chunk}")
```

### 方法3：按 Markdown 标题切分

```python
# 按 Markdown 标题层级切分，保留结构信息
headers_to_split_on = [
    ("#", "H1"),
    ("##", "H2"),
    ("###", "H3"),
]

md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=headers_to_split_on,
    strip_headers=False  # 保留标题内容
)

md_docs = md_splitter.split_text(sample_text)
for doc in md_docs:
    print(f"\n--- 标题信息: {doc.metadata} ---")
    print(f"内容 ({len(doc.page_content)} 字):\n{doc.page_content[:200]}")

# 如果 chunk 仍然过大，进一步用递归切分
# （Markdown 切分 + 递归切分的组合）
final_splitter = RecursiveCharacterTextSplitter(
    chunk_size=400,
    chunk_overlap=50
)
final_chunks = final_splitter.split_documents(md_docs)
print(f"\n二次切分后：{len(final_chunks)} 个 chunk")
```

### 方法4：基于 Token 的切分（更精确控制 Token 数）

```python
# 基于 Token 数切分（与 LLM 的计费单位一致）
token_splitter = TokenTextSplitter(
    model_name="gpt-4o",
    chunk_size=300,      # 最大 300 个 token
    chunk_overlap=30,    # 重叠 30 个 token
)

chunks = token_splitter.split_text(sample_text)
print(f"Token 切分：{len(chunks)} 个 chunk")

# 验证 token 数
enc = tiktoken.encoding_for_model("gpt-4o")
for i, chunk in enumerate(chunks, 1):
    token_count = len(enc.encode(chunk))
    print(f"Chunk {i}: {token_count} tokens, {len(chunk)} 字符")
```

### 方法5：语义切分（Semantic Chunking，高质量方案）

```python
# pip install langchain-experimental
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

# 语义切分：找到语义不连续的边界作为切分点
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

semantic_splitter = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",  # 百分位数阈值
    breakpoint_threshold_amount=95,          # 切分语义距离最大的 5% 处
)

# 注意：语义切分每个句子都会调用 Embedding API，成本较高
semantic_chunks = semantic_splitter.split_text(sample_text)
print(f"语义切分：{len(semantic_chunks)} 个 chunk")
for i, chunk in enumerate(semantic_chunks, 1):
    print(f"\nChunk {i} ({len(chunk)} 字):\n{chunk[:150]}...")
```

### 方法6：代码文件切分

```python
from langchain.text_splitter import PythonCodeTextSplitter, Language

# Python 代码切分（按函数/类切分）
python_splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=1000,
    chunk_overlap=100
)

python_code = '''
import numpy as np

def fibonacci(n: int) -> int:
    """计算斐波那契数列的第 n 项"""
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

class DataProcessor:
    """数据处理类"""

    def __init__(self, data: list):
        self.data = np.array(data)

    def normalize(self) -> np.ndarray:
        """归一化数据到 [0, 1] 范围"""
        min_val = self.data.min()
        max_val = self.data.max()
        return (self.data - min_val) / (max_val - min_val)

    def statistics(self) -> dict:
        """计算基本统计信息"""
        return {
            "mean": float(self.data.mean()),
            "std": float(self.data.std()),
            "min": float(self.data.min()),
            "max": float(self.data.max()),
        }
'''

code_chunks = python_splitter.split_text(python_code)
print(f"代码切分：{len(code_chunks)} 个 chunk")
for i, chunk in enumerate(code_chunks, 1):
    print(f"\nChunk {i}:\n{chunk}")
```

### 实际项目：批量处理文档

```python
from langchain_community.document_loaders import (
    PyMuPDFLoader, UnstructuredWordDocumentLoader,
    TextLoader, WebBaseLoader
)
from langchain.text_splitter import RecursiveCharacterTextSplitter
from pathlib import Path


def load_and_split_documents(doc_paths: list[str], chunk_size: int = 500) -> list:
    """批量加载并切分文档"""

    all_documents = []

    for path in doc_paths:
        path_obj = Path(path)
        try:
            if path_obj.suffix == ".pdf":
                loader = PyMuPDFLoader(path)
            elif path_obj.suffix in [".docx", ".doc"]:
                loader = UnstructuredWordDocumentLoader(path)
            elif path_obj.suffix in [".txt", ".md"]:
                loader = TextLoader(path, encoding="utf-8")
            else:
                print(f"不支持的文件类型：{path}")
                continue

            docs = loader.load()
            all_documents.extend(docs)
            print(f"加载成功：{path}（{len(docs)} 页）")

        except Exception as e:
            print(f"加载失败：{path}，错误：{e}")

    # 切分
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=chunk_size,
        chunk_overlap=int(chunk_size * 0.1),
        separators=["\n\n", "\n", "。", "！", "？", "；", "，", " ", ""],
    )

    chunks = splitter.split_documents(all_documents)

    # 添加 chunk 编号元数据
    for i, chunk in enumerate(chunks):
        chunk.metadata["chunk_id"] = i
        chunk.metadata["chunk_size"] = len(chunk.page_content)

    print(f"\n总计：{len(all_documents)} 个文档 → {len(chunks)} 个 chunk")
    print(f"平均 chunk 大小：{sum(len(c.page_content) for c in chunks) / len(chunks):.0f} 字符")

    return chunks
```

## 常见用法

### 切分策略选择决策树

```
文档类型是什么？
├── Markdown 文档 → MarkdownHeaderTextSplitter + 递归切分
├── 代码文件 → Language-specific splitter（按函数/类）
├── 结构化 HTML → HTMLHeaderTextSplitter
├── PDF/Word 通用文档 → RecursiveCharacterTextSplitter
└── 高质量要求 → SemanticChunker（成本更高）

chunk_size 如何选择？
├── 文档内容简短（FAQ/Q&A） → 100-200
├── 通用知识库 → 300-500（推荐默认值）
├── 详细技术文档 → 500-800
└── 法律/合同长文档 → 800-1200

overlap 设置建议：
└── 设为 chunk_size 的 10%（最小 20，最大 200）
```

### 切分质量评估

```python
def evaluate_chunks(chunks: list) -> dict:
    """评估切分质量"""
    sizes = [len(chunk.page_content) for chunk in chunks]

    stats = {
        "total_chunks": len(chunks),
        "avg_size": sum(sizes) / len(sizes),
        "min_size": min(sizes),
        "max_size": max(sizes),
        "too_small": sum(1 for s in sizes if s < 50),   # 过短的 chunk
        "too_large": sum(1 for s in sizes if s > 1000), # 过长的 chunk
    }

    if stats["too_small"] > len(chunks) * 0.1:
        print(f"警告：{stats['too_small']} 个 chunk 过短（<50字），可能影响检索质量")
    if stats["too_large"] > len(chunks) * 0.05:
        print(f"警告：{stats['too_large']} 个 chunk 过长（>1000字），考虑减小 chunk_size")

    return stats
```

## 小结

- **RecursiveCharacterTextSplitter** 是最推荐的通用切分器，能根据分隔符层级智能切分
- **MarkdownHeaderTextSplitter** 适合结构化文档，保留了标题层级元数据，便于后续过滤
- **语义切分（SemanticChunker）** 质量最高，但会调用 Embedding API，成本较高，适合对准确性要求极高的场景
- **chunk_size 和 overlap** 是最重要的超参数，建议从 chunk_size=500, overlap=50 开始，根据评估结果调整
- **中文文档**建议在分隔符中加入中文标点（`。！？；，`），避免按英文标点切分导致语义断裂
- 2024-2025 年趋势：**Parent Document Retriever**（父文档检索）模式——用小 chunk 索引，检索到后返回对应的大 chunk，兼顾检索精度和上下文完整性
