# 向量检索

## 概念介绍

向量检索（Vector Search）是在高维向量空间中，给定一个查询向量，找出与其最相似的 K 个向量的技术。在 RAG 系统中，向量检索是核心的信息检索手段：将用户问题编码为向量，然后在预先建立的文档向量库中快速找到最相关的文档片段。

向量检索的核心挑战：
- **精确最近邻（Exact NN）**：暴力计算所有向量的相似度，精确但慢，仅适合小规模数据
- **近似最近邻（ANN）**：以少量精度损失换取极大速度提升，适合大规模生产环境

## 核心原理

### 相似度度量

**余弦相似度（Cosine Similarity）**：最常用，衡量两向量夹角：

$$\text{cosine}(A, B) = \frac{A \cdot B}{\|A\| \|B\|}$$

- 范围 [-1, 1]，1 表示完全相同方向，-1 表示完全相反
- 不受向量长度影响，专注于方向（语义）
- 如果向量已归一化，余弦相似度 = 内积（点积）

**欧式距离（L2 Distance）**：

$$d(A, B) = \sqrt{\sum_{i=1}^{n}(A_i - B_i)^2}$$

- 距离越小越相似
- 受向量大小影响

**内积（Dot Product）**：
$$A \cdot B = \sum_{i=1}^{n} A_i B_i$$

- 对已归一化的向量，内积 = 余弦相似度
- OpenAI 的 Embedding 推荐使用余弦相似度

### ANN（近似最近邻）算法

| 算法 | 原理 | 特点 | 代表实现 |
|------|------|------|----------|
| HNSW | 分层小世界图 | 速度极快，内存占用大 | FAISS, Qdrant |
| IVF | 倒排文件索引 | 内存友好，精度略低 | FAISS |
| IVF+PQ | 倒排 + 乘积量化 | 压缩存储，速度快 | FAISS |
| LSH | 局部敏感哈希 | 简单快速 | 早期方案 |
| ScaNN | 各向异性量化 | Google 内部使用 | ScaNN |
| DiskANN | 基于磁盘的索引 | 超大规模数据集 | DiskANN |

**HNSW（Hierarchical Navigable Small World）** 是目前最流行的 ANN 算法：
- 构建分层图结构，越高层越稀疏
- 查询时从顶层开始贪心搜索，逐层细化
- 典型速度：百万级向量，查询 < 1ms

### 混合检索（Hybrid Search）

单纯向量检索存在局限：精确关键词匹配（如专有名词、型号）效果差。

**混合检索 = 向量检索（稠密） + BM25 检索（稀疏）**

- **BM25**：经典词汇检索，基于词频/逆文档频率，擅长精确匹配
- **融合策略**：RRF（倒数排名融合），归一化后加权求和

$$\text{RRF}(d) = \sum_{r \in R} \frac{1}{k + r(d)}$$

其中 $k=60$，$r(d)$ 是文档 $d$ 在各检索结果中的排名。

## 代码示例（Python）

### FAISS 基础使用

```python
import faiss
import numpy as np
from openai import OpenAI

client = OpenAI()

# 获取 Embedding
def get_embeddings(texts: list[str], model: str = "text-embedding-3-small") -> np.ndarray:
    response = client.embeddings.create(model=model, input=texts)
    embeddings = [item.embedding for item in sorted(response.data, key=lambda x: x.index)]
    return np.array(embeddings, dtype=np.float32)

# 示例文档库
documents = [
    "RAG 是检索增强生成技术，能有效减少 LLM 的幻觉问题",
    "FAISS 是 Facebook 开发的高效向量相似度搜索库",
    "向量数据库专门优化了向量的存储和检索性能",
    "BM25 是一种基于词频的经典信息检索算法",
    "Embedding 模型将文本映射到高维向量空间",
    "Transformer 架构是现代大语言模型的基础",
    "Python 是目前最流行的 AI 开发语言",
    "机器学习通过数据训练使计算机自动学习规律",
]

# 构建向量索引
doc_embeddings = get_embeddings(documents)
dimension = doc_embeddings.shape[1]
print(f"向量维度: {dimension}, 文档数: {len(documents)}")

# 方法1：精确搜索（Flat Index，暴力计算）
index_flat = faiss.IndexFlatIP(dimension)  # IP = Inner Product（内积）
# 归一化使内积 = 余弦相似度
faiss.normalize_L2(doc_embeddings)
index_flat.add(doc_embeddings)

# 查询
query = "如何减少大模型的幻觉问题？"
query_embedding = get_embeddings([query])
faiss.normalize_L2(query_embedding)

scores, indices = index_flat.search(query_embedding, k=3)
print("\n精确检索结果（Flat Index）:")
for score, idx in zip(scores[0], indices[0]):
    print(f"  相似度: {score:.4f} | {documents[idx]}")
```

### FAISS HNSW 索引（生产级别）

```python
import faiss
import numpy as np

def build_hnsw_index(embeddings: np.ndarray, M: int = 32) -> faiss.IndexHNSWFlat:
    """
    构建 HNSW 索引
    M: 每个节点的最大连接数，M 越大，精度越高，内存越大（通常 16-64）
    """
    dimension = embeddings.shape[1]
    # HNSW + 内积（适配余弦相似度）
    index = faiss.IndexHNSWFlat(dimension, M, faiss.METRIC_INNER_PRODUCT)

    # 构建参数（影响索引质量，越大越精确但构建越慢）
    index.hnsw.efConstruction = 200
    # 查询参数（影响检索质量，越大越精确但越慢）
    index.hnsw.efSearch = 100

    # 归一化后添加
    embeddings_copy = embeddings.copy()
    faiss.normalize_L2(embeddings_copy)
    index.add(embeddings_copy)

    return index


def build_ivf_index(embeddings: np.ndarray, nlist: int = 100) -> faiss.IndexIVFFlat:
    """
    构建 IVF 索引（适合大规模数据，内存友好）
    nlist: 聚类中心数，通常设为 sqrt(N)
    """
    dimension = embeddings.shape[1]

    # 量化器（用于 IVF）
    quantizer = faiss.IndexFlatIP(dimension)
    index = faiss.IndexIVFFlat(quantizer, dimension, nlist, faiss.METRIC_INNER_PRODUCT)

    embeddings_copy = embeddings.copy()
    faiss.normalize_L2(embeddings_copy)

    # IVF 需要先训练
    index.train(embeddings_copy)
    index.add(embeddings_copy)

    # 查询时探索的聚类数（nprobe 越大，精度越高但越慢）
    index.nprobe = 10

    return index


# 构建 HNSW 索引
hnsw_index = build_hnsw_index(doc_embeddings, M=32)
print(f"HNSW 索引文档数: {hnsw_index.ntotal}")

# 保存和加载索引
faiss.write_index(hnsw_index, "hnsw_index.bin")
loaded_index = faiss.read_index("hnsw_index.bin")

# 查询
query_emb = get_embeddings(["什么是向量数据库？"])
faiss.normalize_L2(query_emb)
scores, indices = loaded_index.search(query_emb, k=3)
print("\nHNSW 检索结果:")
for score, idx in zip(scores[0], indices[0]):
    print(f"  相似度: {score:.4f} | {documents[idx]}")
```

### 混合检索（向量 + BM25）

```python
from rank_bm25 import BM25Okapi
import jieba
import numpy as np
from openai import OpenAI

client = OpenAI()

class HybridRetriever:
    """混合检索器：向量检索 + BM25，使用 RRF 融合"""

    def __init__(self, documents: list[str], embedding_model: str = "text-embedding-3-small"):
        self.documents = documents
        self.embedding_model = embedding_model

        # 构建 BM25 索引
        tokenized_docs = [list(jieba.cut(doc)) for doc in documents]
        self.bm25 = BM25Okapi(tokenized_docs)

        # 构建 FAISS 索引
        import faiss
        embeddings = self._get_embeddings(documents)
        faiss.normalize_L2(embeddings)
        self.dimension = embeddings.shape[1]
        self.index = faiss.IndexFlatIP(self.dimension)
        self.index.add(embeddings)
        self.faiss = faiss

    def _get_embeddings(self, texts: list[str]) -> np.ndarray:
        response = client.embeddings.create(model=self.embedding_model, input=texts)
        return np.array([item.embedding for item in sorted(response.data, key=lambda x: x.index)], dtype=np.float32)

    def rrf_score(self, ranks: list[int], k: int = 60) -> float:
        """RRF 得分：倒数排名融合"""
        return sum(1 / (k + r + 1) for r in ranks)

    def search(self, query: str, top_k: int = 5, alpha: float = 0.5) -> list[dict]:
        """
        混合检索
        alpha: 向量检索权重（0=纯BM25，1=纯向量）
        """
        n = len(self.documents)

        # 向量检索
        query_emb = self._get_embeddings([query])
        self.faiss.normalize_L2(query_emb)
        dense_scores, dense_indices = self.index.search(query_emb, k=n)
        dense_rank = {idx: rank for rank, idx in enumerate(dense_indices[0])}

        # BM25 检索
        query_tokens = list(jieba.cut(query))
        bm25_scores = self.bm25.get_scores(query_tokens)
        bm25_rank = {idx: rank for rank, idx in enumerate(np.argsort(bm25_scores)[::-1])}

        # RRF 融合
        rrf_scores = {}
        for idx in range(n):
            dense_r = dense_rank.get(idx, n)
            sparse_r = bm25_rank.get(idx, n)
            rrf_scores[idx] = alpha * self.rrf_score([dense_r]) + (1 - alpha) * self.rrf_score([sparse_r])

        # 按 RRF 得分排序
        sorted_indices = sorted(rrf_scores.keys(), key=lambda x: rrf_scores[x], reverse=True)

        results = []
        for idx in sorted_indices[:top_k]:
            results.append({
                "document": self.documents[idx],
                "rrf_score": rrf_scores[idx],
                "dense_score": float(dense_scores[0][dense_rank.get(idx, -1)]) if idx in dense_rank else 0,
                "bm25_score": float(bm25_scores[idx]),
            })
        return results


# 使用
retriever = HybridRetriever(documents)
results = retriever.search("FAISS 如何检索相似向量？", top_k=3)
for r in results:
    print(f"RRF: {r['rrf_score']:.4f} | 内容: {r['document']}")
```

### 使用 Qdrant 向量数据库

```python
from qdrant_client import QdrantClient
from qdrant_client.models import (
    Distance, VectorParams, PointStruct,
    Filter, FieldCondition, MatchValue,
    SearchRequest
)
from openai import OpenAI
import uuid

client_oai = OpenAI()
qdrant = QdrantClient(":memory:")  # 内存模式，生产使用 host="localhost"

COLLECTION_NAME = "knowledge_base"
VECTOR_DIM = 1536

# 创建集合
qdrant.create_collection(
    collection_name=COLLECTION_NAME,
    vectors_config=VectorParams(
        size=VECTOR_DIM,
        distance=Distance.COSINE
    )
)

# 插入文档
def add_documents(texts: list[str], metadatas: list[dict] = None):
    response = client_oai.embeddings.create(
        model="text-embedding-3-small",
        input=texts
    )
    embeddings = [item.embedding for item in sorted(response.data, key=lambda x: x.index)]

    points = []
    for i, (text, emb) in enumerate(zip(texts, embeddings)):
        metadata = metadatas[i] if metadatas else {}
        metadata["text"] = text
        points.append(PointStruct(
            id=str(uuid.uuid4()),
            vector=emb,
            payload=metadata
        ))

    qdrant.upsert(collection_name=COLLECTION_NAME, points=points)
    print(f"成功插入 {len(points)} 个文档")


# 添加带元数据的文档
add_documents(
    texts=documents,
    metadatas=[
        {"source": "doc1", "category": "RAG", "date": "2024-01-01"},
        {"source": "doc2", "category": "AI", "date": "2024-01-02"},
    ] * 4
)

# 相似度搜索
def search_documents(query: str, top_k: int = 3, filter_category: str = None):
    query_emb = client_oai.embeddings.create(
        model="text-embedding-3-small",
        input=query
    ).data[0].embedding

    # 可选元数据过滤
    query_filter = None
    if filter_category:
        query_filter = Filter(
            must=[FieldCondition(
                key="category",
                match=MatchValue(value=filter_category)
            )]
        )

    results = qdrant.search(
        collection_name=COLLECTION_NAME,
        query_vector=query_emb,
        limit=top_k,
        query_filter=query_filter,
        with_payload=True
    )

    return [{"text": r.payload["text"], "score": r.score, "metadata": r.payload} for r in results]


results = search_documents("向量检索是如何工作的？", top_k=3)
for r in results:
    print(f"相似度: {r['score']:.4f} | {r['text']}")
```

### 检索性能对比

```python
import time
import numpy as np

def benchmark_search(index, query_vectors: np.ndarray, k: int = 5, n_trials: int = 100):
    """测试检索性能"""
    times = []
    for _ in range(n_trials):
        q = query_vectors[np.random.randint(len(query_vectors))].reshape(1, -1)
        start = time.perf_counter()
        index.search(q, k)
        times.append(time.perf_counter() - start)

    return {
        "avg_ms": np.mean(times) * 1000,
        "p99_ms": np.percentile(times, 99) * 1000,
        "qps": 1 / np.mean(times)
    }
```

### 向量数据库选型

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| FAISS | 纯 Python/C++，轻量，无持久化 | 原型开发，研究 |
| Chroma | 开源，易用，支持持久化 | 中小型项目 |
| Qdrant | Rust 实现，高性能，过滤能力强 | 生产环境首选 |
| Milvus | 分布式，企业级，云原生 | 大规模企业场景 |
| Weaviate | 支持多模态，GraphQL API | 复杂查询场景 |
| pgvector | PostgreSQL 扩展 | 已有 PG 的项目 |

## 小结

- **余弦相似度**是文本向量检索的标准度量，对已归一化向量等同于内积计算
- **HNSW** 是目前最流行的 ANN 算法，速度极快，推荐用于生产环境
- **混合检索（Hybrid Search）= 向量检索 + BM25**，通过 RRF 融合能显著提升检索质量，是 2024-2025 年 RAG 的标准实践
- **Qdrant** 在生产环境中表现优秀，支持元数据过滤、持久化、分布式，是当前最受欢迎的向量数据库
- **元数据过滤**与向量检索结合，能有效缩小检索范围，提升精度（如按时间、来源、类别过滤）
- 实际项目建议：**FAISS 用于离线批量计算**，**Qdrant/Milvus 用于在线实时服务**
